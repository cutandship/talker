# 02. Command Mode — голосовое редактирование выделенного текста

> Выделил текст → отдельный хоткей → «переписать формально / в маркированный список / перевести на английский» → LLM правит inline.

**Категория:** Tier 1 — главная фича-отличие Wispr Flow / Superwhisper / AquaVoice.
**Готовность концепта:** 🟢 High.

---

## Зачем

Сейчас Talker — это **dictation** (голос → текст). Command Mode превращает его в **voice editor**: ты говоришь не текст, а **инструкцию** к существующему тексту.

Сценарии:
- В письме выделил абзац, сказал "сделай более коротко" → абзац переписан на месте.
- В переписке выделил черновик, сказал "переведи на английский" → готово.
- В коде выделил функцию, сказал "добавь docstring" → docstring появился.
- Без выделения: сказал "напиши извинение что опоздаю" → текст вставлен в курсор.

Использует уже существующую LLM-цепочку, новый только промпт и режим.

---

## Технический подход

### Хоткей

- Дефолт: `Ctrl+Alt+Space` уже занят continuous-режимом. Перенесём continuous на `Ctrl+Shift+Space`, оставим `Ctrl+Alt+Space` (или новый `right ctrl`) для Command Mode.
- Лучше — конфигурируемый, через те же поля настроек.

### Алгоритм

1. **Жмём Command Mode хоткей.**
2. **Читаем выделение** через clipboard hack:
   - Сохраняем текущий clipboard.
   - Шлём `Ctrl+C`.
   - Ждём 80 мс, читаем `pyperclip.paste()`.
   - Если содержимое clipboard **изменилось** — это выделение. Если нет — выделения нет.
   - Восстанавливаем clipboard.
3. **Записываем команду голосом** (стандартный push-to-talk pipeline, без cleanup).
4. **Транскрибируем команду** через Whisper.
5. **Формируем промпт:**
   - Если есть выделение: `SYSTEM` = command-mode system prompt, `USER` = `Инструкция: {command}\n\nТекст: {selection}`.
   - Если нет выделения: `USER` = `{command}` (свободная генерация).
6. **Зовём LLM** через ту же цепочку cleaners (`build_cleaner_chain` уже умеет fallback).
7. **Вставляем результат:**
   - Если было выделение: clipboard + `Ctrl+V` (заменит выделение).
   - Если не было: вставка в курсор как обычно.
8. **Bubble показывает** результат как safety net (если вставка не сработала).

### Промпт для Command Mode (черновик)

```
Ты — voice-driven редактор. На вход поступают:
1. Инструкция от пользователя (произнесённая вслух).
2. (Опционально) текст, к которому инструкция применяется.

Если текст дан — применяй инструкцию к нему и верни ТОЛЬКО результат.
Если текста нет — выполняй инструкцию свободно (генерация).

Сохраняй:
- язык оригинала, если не сказано иначе;
- технические термины, имена собственные, URL'ы — без изменений.

Не добавляй пояснений, заголовков, кавычек. Только сам отредактированный или сгенерированный текст.
```

### State machine

Новое состояние `State.COMMAND_RECORDING` + `State.COMMAND_PROCESSING`. В FlowBar показывать особым цветом (фиолетовый/cyan) и иконкой ⌘ или ✎.

## Архитектура

**Новые модули:** нет, реюз `cleaner.py`.

**Изменения:**
- `main.py`:
  - Новый хоткей-хендлер `_on_command_press` / `_on_command_release`.
  - Метод `_capture_selection() -> str | None`.
  - Метод `_process_command(audio, selection)`.
  - Новый State enum value.
- `cleaner.py`:
  - Метод `CleanerChain.run(system_prompt: str, user_msg: str) -> tuple[str, bool]`. Уже близко по API — расширить, чтобы system был параметром.
- `config.py`:
  - `CommandModeConfig { hotkey: str = "ctrl+alt+space", system_prompt: str = "..." }`.
- `ui.py`:
  - В Настройках — секция «Command Mode»: чекбокс, хоткей, кастомный system prompt (textarea), модель (можно отдельную, дешёвую, не очистка).
  - FlowBar — цвет/лейбл для command-states.

## Edge cases

| Сценарий | Поведение |
|---|---|
| Clipboard hack не сработал (приложение блокирует Ctrl+C) | Считаем что выделения нет. В bubble сообщить «не удалось прочитать выделение». |
| Выделение огромное (>20k символов) | Обрезаем до N токенов с предупреждением в bubble. |
| LLM ответ — пустая строка | Не вставляем. Bubble: «модель вернула пусто». |
| Юзер передумал в середине записи | Esc → cancel, ничего не делаем. |
| Команда не распознана как инструкция, выглядит как обычная диктовка | Это ОК — модель просто перепишет текст «по смыслу», по сути выполнит «улучши». Лечение: чёткий system prompt + примеры few-shot. |
| Конфликт с push-to-talk хоткеем | Хоткей валидируется в Настройках, при пересечении — предупреждение. |

## Acceptance criteria

- Включаю Command Mode, выделяю абзац, говорю «короче», отпускаю — абзац заменяется на короткую версию.
- Включаю без выделения, говорю «напиши хайку про осень» — текст появляется в курсоре.
- Команда занимает ≤ 3 сек после отпускания клавиши на small + быстрой LLM.
- При недоступности LLM — bubble с raw командой и сообщением «cleanup недоступен».
- Не ломает существующий push-to-talk pipeline.

## Открытые вопросы

- Использовать ту же LLM-цепочку (`cleaners`) или отдельный список «command-mode cleaners»? Дефолт: тот же, проще. В будущем — отдельный.
- Stream Command Mode ответ или ждать целиком? Дефолт: ждать (короткие тексты).
- Show preview перед вставкой? Дефолт: нет (медленнее workflow). Опционально через настройку.
