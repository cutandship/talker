# 33. Надёжная вставка: восстановление буфера, «вставить последний транскрипт», скретчпад, per-app таблица

> Закрыть самые частые бытовые сбои потока: гарантированно восстанавливать буфер обмена, дать сеть безопасности при неудачной вставке («вставить последний транскрипт» + скретчпад) и спец-обработку приложений, которые ломают clipboard-вставку (терминалы, Outlook classic, RDP).

**Категория:** UX потока ввода — дёшево, высокая ежедневная отдача.
**Эффект:** 🔥 высокая отдача, низкая стоимость.
**Готовность концепта:** 🟢 High.
**Источник идеи:** [research/2026-05-29](research/2026-05-29_what-to-add-next.md) F5 (Wispr Flow).

---

## Зачем

У Talker уже есть каскад вставки UIA → SendInput → clipboard+Ctrl+V ([injector.py](../injector.py)) и защита password-полей. Wispr Flow (прямой ориентир) добавляет то, чего не хватает для надёжного ежедневного потока:
1. **Гарантированное восстановление буфера** после вставки (текст пользователя в буфере не затирается).
2. **Сеть безопасности**, если авто-вставка не сработала: хоткей «вставить последний транскрипт» (у Wispr — дефолт Alt+Shift+Z) и **скретчпад** — надиктованное не теряется.
3. **Per-app обработка** приложений, систематически ломающих clipboard-вставку: терминалы (cmd/PowerShell/Wezterm) → `Ctrl+Shift+V`; **Outlook classic** → задержка 1–2 с; Citrix/RDP/виртуалки.

---

## Технический подход

### Restore буфера

`OutputConfig.restore_clipboard` уже есть ([config.py:107](../config.py)). Убедиться, что: сохраняем прежний буфер → кладём текст → Ctrl+V → через ~500 мс возвращаем прежний (при отмене — не возвращаем). Сделать таймаут восстановления надёжным (поток/таймер).

### Сеть безопасности

- **«Вставить последний транскрипт»** — хоткей, который повторно вставляет последний результат из истории ([history_mgr.py](../history_mgr.py) уже хранит записи). Срабатывает, если авто-вставка ушла «не туда» или не сработала (см. guard `_paste` → `skipped_typing`, [main.py:865](../main.py)).
- **Скретчпад** — маленькое окно/буфер, куда всегда падает последний(е) транскрипт(ы); открывается хоткеем. opt-in.

### Per-app таблица вставки

Использовать `modes.get_foreground_info()` ([modes.py:70](../modes.py)) — по классу/процессу окна выбрать метод/задержку:

```
терминалы (cmd/powershell/WindowsTerminal/wezterm) → Ctrl+Shift+V
OUTLOOK.EXE (classic)                              → задержка 1–2 с перед Ctrl+V
Citrix / RDP / виртуалки                           → SendInput Unicode (минуя clipboard)
```

Встроить в `injector.inject` как выбор стратегии по foreground-инфо.

## Архитектура

- `injector.py` — надёжный restore буфера; per-app выбор метода/задержки по foreground-инфо.
- `main.py` — хоткей «paste last transcript» (берёт из `history`); опц. скретчпад-окно.
- `config.py` — `[output]`: `paste_last_hotkey`, `scratchpad_hotkey`; (опц.) per-app overrides.
- `modes.py` — переиспользуется для детекта приложения.
- `ui.py` — настройка хоткеев + список проблемных приложений.

## Зависимости

Нет (всё на существующих keyboard/pyperclip/UIA).

## Edge cases

| Сценарий | Поведение |
|---|---|
| Пользователь печатает в момент вставки | Уже есть `_TYPING_GUARD_SEC` → bubble; «paste last» закрывает этот случай. |
| Буфер с картинкой/файлами | Restore сохранять любой формат буфера, не только текст. |
| Терминал не принимает Ctrl+Shift+V | Fallback на SendInput-набор. |
| Скретчпад растёт | Хранить N последних, чистить по retention (как история). |
| Outlook classic «съел» вставку | Задержка перед Ctrl+V; при неудаче — bubble + «paste last». |

---

## Acceptance criteria

- После вставки прежнее содержимое буфера восстановлено (текст/не-текст).
- Хоткей «вставить последний транскрипт» работает в любой момент.
- В терминале вставка идёт через Ctrl+Shift+V; в Outlook classic — с задержкой; промахов меньше.
- Надиктованное не теряется даже при неудачной авто-вставке.
