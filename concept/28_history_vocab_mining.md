# 28. Майнинг истории → авто-предложения в словарь

> Talker периодически просматривает историю расшифровок, находит часто встречающиеся имена/термины, которых ещё нет в словаре, и предлагает добавить их одной кнопкой. Словарь наполняется без ручного труда — пассивно, в дополнение к bubble-правкам (концепт 06).

**Категория:** Tier 3 — zero-effort наполнение словаря.
**Эффект:** 👍 хорошо.
**Готовность концепта:** 🟢 High.

---

## Зачем

Концепт 06 (auto-learning) учится только когда юзер нажал «✎ Поправить» в bubble ([main.py:916](../main.py)) — это явное действие, происходит редко. Большинство терминов так и не попадают в словарь, и Whisper продолжает их путать.

Пассивный майнинг закрывает разрыв: в `history.json` ([history_mgr.py](../history_mgr.py)) уже лежат сотни реальных расшифровок. Если слово-имя встречается там часто, а в словаре его нет — почти наверняка это рабочий термин юзера. Предлагаем добавить.

---

## Технический подход

### Сбор кандидатов

Переиспользуем уже готовую эвристику `_looks_like_name` ([vocabulary.py:112](../vocabulary.py)) и `_STOPWORDS` ([vocabulary.py:68](../vocabulary.py)):

```python
from collections import Counter
from vocabulary import _looks_like_name, normalize_words

def mine_candidates(entries, existing_vocab, min_count=3, top=15):
    existing = {w.lower() for w in existing_vocab}
    counter = Counter()
    for e in entries:                          # history_mgr.entries()
        for tok in e["text"].split():
            w = tok.strip(" .,;:!?\"'«»()[]{}…")
            if _looks_like_name(w) and w.lower() not in existing:
                counter[w] += 1
    return [w for w, n in counter.most_common(top) if n >= min_count]
```

`_looks_like_name` уже отсекает стоп-слова, цифры, короткие слова и оставляет имена собственные / латиницу / camelCase / слова с точкой — ровно то, что полезно в словаре.

### Когда запускать

- По кнопке «Предложить слова из истории» в Настройках (явный, безопасный путь — как первый шаг).
- Опционально: раз в N запусков показывать tray-уведомление «Talker заметил 5 новых терминов — добавить?».

### UI подтверждения

Диалог со списком кандидатов и чекбоксами (по умолчанию все отмечены) → отмеченные уходят в `config.vocabulary.words`, дальше штатный путь: `save_config` + `_apply_runtime_overrides` ([main.py:934](../main.py)) → словарь мгновенно у Whisper.

Это **предложение**, а не авто-добавление: эвристика шумновата (может предложить случайное капитализированное слово), решает юзер. Отклонённые — в blacklist, чтобы не предлагать снова.

## Edge cases

| Сценарий | Поведение |
|---|---|
| Пустая / короткая история | Кандидатов нет, кнопка сообщает «недостаточно данных». |
| Капитализированное начало предложения («Привет») | `min_count≥3` + частотный порог отсекают разовый шум; стоп-слова уже в `_STOPWORDS`. |
| Юзер отклонил слово | В blacklist — больше не предлагаем. |
| Слово уже в словаре | Отфильтровано (`existing`). |
| История очищается (retention, концепт 15) | Майнинг работает по тому, что есть; не критично. |
| Опечатка-артефакт STT попала в топ | Юзер просто не отмечает её в диалоге. |

---

## Acceptance criteria

- Кнопка показывает реальные частые термины из истории, которых нет в словаре.
- Отмеченные добавляются и сразу применяются к Whisper (без перезагрузки модели).
- Отклонённые не предлагаются повторно.
- Стоп-слова и обычная лексика в кандидаты не попадают.

## Открытые вопросы

- Авто-добавлять при очень высокой частоте (например, ≥10 повторов) или всегда спрашивать? → всегда спрашивать в v1; авто — опционально позже.
