# 24. Нормализация чисел, дат и единиц (ITN)

> Inverse Text Normalization: «двадцать пять процентов» → «25 %», «две тысячи двадцать шестой» → «2026», «в три часа дня» → «15:00». Главная боль русского Whisper — он пишет числительные словами.

**Категория:** Tier 2 — очень заметно на деловой/технической речи.
**Эффект:** 🔥 круто.
**Готовность концепта:** 🟡 Medium (промпт-путь готов; локальные правила требуют бенчмарка).

---

## Зачем

Whisper транскрибирует числа прописью: «у нас выросло на двадцать пять процентов за два квартала». Читать и редактировать это неудобно, а ни один текущий шаг пайплайна ITN не делает — `smart_format` ([main.py:879](../main.py)) и `cursor_format.py` чинят только регистр/пробел у курсора, не числа.

Хочется: `25 %`, `2026`, `15:00`, `1 500 ₽`, `№ 3`, `п. 2.1`.

---

## Технический подход

Два пути, **взаимодополняющих** — выбираются по тому, есть ли в цепочке LLM.

### Путь A (основной) — инструкция в промпт очистки

Если в цепочке есть LLM-бэкенд, ITN почти бесплатен: дописываем в `_SYSTEM_PROMPT` ([cleaner.py:12](../cleaner.py)) блок (через тот же механизм, что глоссарий из концепта 23, либо отдельным `output.number_format`-флагом):

```
- числа, проценты, даты, время и денежные суммы записывай цифрами
  («двадцать пять процентов» → «25 %», «две тысячи двадцать шестой год» → «2026 год»),
  если это не нарушает смысл; телефоны и номера оставляй как произнесено.
```

Плюсы: ноль зависимостей, понимает контекст («три» как число vs «три» как глагол). Минусы: не работает, когда очистка = noop/punctuation (текущий дефолт `config.toml`).

### Путь B (локальный) — модуль `itn.py`

Детерминированный rule-based проход для офлайна / без LLM. Применяется к финальному тексту. Покрываем частотное ядро:

- кардиналы 0–999 999 (составные: «двадцать пять» → 25);
- проценты: `процент(ов/а)` после числа → ` %`;
- годы-порядковые: «две тысячи двадцать шестой (год)» → «2026»;
- время: «в три часа (дня/вечера)» → «15:00» (с поправкой на дня/вечера/ночи/утра);
- разделение тысяч пробелом (1 500).

Кандидаты для оценки (выбрать на бенчмарке, не тащить вслепую):
`text-to-num` / `WeTextProcessing` (есть ru-граммары) / собственный мини-парсер.

```python
# itn.py — скелет
def normalize(text: str, lang: str = "ru") -> str:
    if not lang.startswith("ru"):
        return text
    text = _cardinals(text)     # числительные → цифры
    text = _percent(text)       # "<n> процентов" → "<n> %"
    text = _years(text)         # порядковые-годы
    text = _clock(text)         # "в три часа дня" → "15:00"
    return text
```

### Где в пайплайне (для пути B)

В `_process` ([main.py:805](../main.py)) **после** очистки, **до** `_paste`:

```python
if self.config.output.number_format:
    from itn import normalize
    text = normalize(text, self.config.stt.language)
```

Так и LLM-путь (A), и локальный (B) дают цифры; если оба включены — B лишь «дочистит» остатки.

## Архитектура

**Новый модуль:** `itn.py` (путь B).

**Изменения:**
- `config.py` — флаг `OutputConfig.number_format: bool = True` ([config.py:106](../config.py)), запись в `save_config`.
- `cleaner.py` / `vocabulary.py` — добавить ITN-предложение в собираемый промпт (путь A), под тем же флагом.
- `main.py` — вызов `itn.normalize` в `_process` после очистки.
- `ui.py` — чекбокс «Числа цифрами (25 %, 2026)» в `SettingsWindow`.

---

## Зависимости

- Путь A: нет.
- Путь B: либо нулевые (свои правила), либо одна из ITN-библиотек после бенчмарка.

---

## Edge cases

| Сценарий | Поведение |
|---|---|
| «три» как глагол («не три глаза») | Путь A (LLM) различает по контексту; путь B — только числовые позиции, риск есть → консервативные правила. |
| Номера телефонов / коды | Не трогать (в промпте оговорено; в правилах — не склеивать длинные цепочки). |
| Код-режим (концепт 07, IDE) | В `code`-mode числа в коде не нормализуем — mode-промпт это уже оговаривает; путь B можно пропускать для `code`. |
| «полтора», «пол-второго» | Редкие дроби — оставить прописью в v1. |
| Десятичные «ноль целых пять» | v2; в v1 не трогаем. |

---

## Acceptance criteria

- «двадцать пять процентов» → «25 %» (хотя бы один из путей включён).
- Годы и время в типовых формах нормализуются.
- Глаголы-омонимы не ломаются на тестовом наборе из 20 фраз.
- Флаг выключает фичу полностью.

---

## Сложность

- Путь A: ~1 час, ~15 LOC (строка промпта + флаг).
- Путь B: ~6–10 часов, ~250 LOC + бенчмарк. Делать после A.

---

## Открытые вопросы

- Пробел перед `%` (типографика РФ — да, `25 %`) или слитно? → флаг/дефолт `25 %`.
- Денежные суммы и валюты (₽/$/€) — рискованно, отдельным под-флагом во второй итерации.
