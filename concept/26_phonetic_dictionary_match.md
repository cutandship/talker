# 26. Фонетический матч для словаря-замены

> Чтобы не вбивать руками все варианты искажения («клод / клот / клауд / клода»), Talker ловит слова, **близкие по звучанию** к каноническому термину, и заменяет их автоматически. Один `Claude` покрывает весь разброс произношения.

**Категория:** Tier 3 — надстройка над концептом 22, снижает ручной труд.
**Эффект:** 👍 хорошо.
**Готовность концепта:** 🟡 Medium — нужен подбор порога, чтобы не заменять лишнее.

---

## Зачем

Словарь-замена (концепт 22) детерминирован, но требует перечислить все варианты. Whisper же искажает имена бесконечно вариативно: `клод, клот, клауд, кload, клода, клот опус`. Перечислять — утомительно и неполно.

Фонетический матч решает: задаёшь канон `Claude` (+ как он звучит по-русски), Talker сам ловит близкие токены по фонетическому коду, а не по буквам.

---

## Технический подход

### Идея

Для каждого канонического термина храним его **произношение** (как слышится по-русски), считаем фонетический ключ; каждый токен сырого STT тоже переводим в ключ; при совпадении ключа (или малом расстоянии) — заменяем на канон.

```toml
[[replacement]]
to       = "Claude"
sounds   = "клод"        # как звучит; из этого берём фонокод
phonetic = true
```

### Фонетический алгоритм

Латинские Soundex/Metaphone на русский не годятся. Варианты (бенчмарк обязателен):

1. **Своя нормализация** (рекомендуется для v1, ноль зависимостей): схлопывание по группам звуков —
   оглушение (`д→т`, `б→п`, `в→ф`), `о/а → a` в безударных, удаление мягкого/твёрдого знаков,
   `кс→x`, двойные → одинарные. `клод`, `клот`, `клауд` → один ключ `клат`.
2. Библиотека `jellyfish` (Metaphone/Damerau-Levenshtein) — латиница; полезна только если канон латинский.
3. Расстояние Дамерау–Левенштейна по нормализованной форме с порогом ≤1–2.

```python
def phonkey(w: str) -> str:
    w = w.lower().replace("ь", "").replace("ъ", "")
    w = w.translate(_VOICED_TO_VOICELESS)     # д→т, б→п, в→ф, г→к, з→с, ж→ш
    w = re.sub(r"(.)\1+", r"\1", w)           # двойные → одинарные
    return w
```

### Применение

В `replacements.py` (концепт 22) добавляем фонетическую ветку: токенизируем текст, для слов длиной ≥4 сверяем `phonkey(token)` с `phonkey(sounds)` всех правил `phonetic=true`; при совпадении и расстоянии ≤ порога — замена с сохранением регистра. Работает **после** точных правил.

## Edge cases

| Сценарий | Поведение |
|---|---|
| Канон случайно похож на частое слово | Порог расстояния + минимальная длина 4; `phonetic` включается пер-правило, не глобально. |
| Два канона с близким звучанием | Берём минимальное расстояние; при равенстве — первый в списке + лог-предупреждение. |
| Короткие слова (≤3) | Не матчим фонетически — слишком много ложных. |
| Ударения меняют гласные | Нормализация безударных `о/а→a`, `е/и→i` снижает чувствительность. |
| Производительность | Индекс ключей строим один раз; на реплику — O(токены × правила), реплики короткие. |

---

## Acceptance criteria

- Канон `Claude` (`sounds=клод`) ловит `клот`, `клауд`, `клода` без их явного перечисления.
- Частые обычные слова не подменяются (тест на 50 фразах без терминов — 0 ложных замен).
- Порог и поведение настраиваются; фича — пер-правило.
