# 23. Глоссарий в промпт очистки

> Список терминов пользователя добавляется в системный промпт LLM-очистки, чтобы модель чинила то, что плоская замена (концепт 22) не ловит — **омофоны по контексту** («компания/кампания», «придел/предел») и непредсказуемые искажения имён.

**Категория:** Tier 2 — копеечное дополнение к словарю, закрывает контекстные ошибки.
**Эффект:** 🔥 круто.
**Готовность концепта:** 🟢 High.

---

## Зачем

Сейчас LLM-чистильщик о терминах пользователя **не знает ничего**: `_SYSTEM_PROMPT` фиксирован ([cleaner.py:12](../cleaner.py)), а словарь уходит только в Whisper. Из-за этого:

- Whisper расслышал «компания», а по смыслу была «кампания» — плоская замена (22) бессильна, нужен контекст.
- Имя исказилось нестандартно («Клот Опус» вместо «Claude Opus») — в словарь-замену все варианты не внесёшь.

LLM в цепочке уже стоит и отлично справляется с такими правками — **если ей сказать**, какие термины канонические. Это буквально одна строка в промпт.

---

## Технический подход

### Сборка суффикса глоссария

Источник — те же `config.vocabulary.words` (+ опционально `to`-значения из правил концепта 22):

```python
def glossary_suffix(words: list[str]) -> str:
    words = [w for w in words if w.strip()]
    if not words:
        return ""
    return (
        "\n\nКанонические написания терминов и имён (используй ровно так, "
        "исправляй искажения по смыслу): " + ", ".join(words) + "."
    )
```

### Проброс в очистку

Проблема: при дефолтном режиме `main.py` передаёт `system_prompt=None`, и `_SYSTEM_PROMPT` подставляется **внутри** `Cleaner.clean` ([cleaner.py:83](../cleaner.py)) — снаружи дописать глоссарий некуда. Решение — отдельный параметр `glossary`, который дописывается к любому используемому промпту:

```python
# cleaner.py
def clean(self, text, system_prompt=None, glossary: str = "") -> str:
    sys = (system_prompt or _SYSTEM_PROMPT) + glossary
    ...
```

`CleanerChain.clean` ([cleaner.py:137](../cleaner.py)) прокидывает `glossary` в каждый бэкенд. В `main.py` ([main.py:813](../main.py)):

```python
text, cleaned = self.cleaner_chain.clean(
    text_in,
    system_prompt=mode_prompt,                 # per-app режим (концепт 07) сохраняется
    glossary=self._glossary_suffix,            # пересобирается при save конфига
)
```

Глоссарий **складывается** с per-app промптом ([main.py:812](../main.py), `_current_mode_prompt`), не заменяет его.

### Бюджет

Глоссарий короткий (десятки слов), на стоимость/латентность очистки влияет незаметно. При очень длинном словаре — обрезать так же, как `build_initial_prompt` (например, первые 80 слов).

## Архитектура

**Изменения:**
- `cleaner.py` — параметр `glossary` в `Cleaner.clean`, всех реализациях (`ApiCleaner`/`OllamaCleaner`/`LocalLlmCleaner`/`NoopCleaner`/`PunctuationCleaner`) и `CleanerChain.clean`. Noop/Punctuation просто игнорируют.
- `main.py` — `self._glossary_suffix = glossary_suffix(cfg.vocabulary.words)`, пересборка в `_on_settings_saved` ([main.py:1164](../main.py)) и `_on_bubble_correction` ([main.py:916](../main.py)); проброс в оба вызова `cleaner_chain.clean` ([main.py:813](../main.py), [main.py:1092](../main.py) — Command Mode).
- Утилиту `glossary_suffix` положить в `vocabulary.py` (рядом с `build_initial_prompt`).

**UI:** ничего нового — переиспользуется секция «Словарь» ([ui.py:2056](../ui.py)).

## Edge cases

| Сценарий | Поведение |
|---|---|
| Очистка = noop / только punctuation (текущий дефолт `config.toml`) | Глоссарий не применяется — LLM нет. Здесь работает концепт 22 (замена). Это и есть причина иметь оба слоя. |
| Пустой словарь | `glossary_suffix` возвращает `""`, промпт не меняется. |
| Command Mode | Тоже передаём глоссарий — имена в свободной генерации полезны. |
| Модель «переусердствовала» и переписала текст под термины | Температура 0.1 ([cleaner.py:102](../cleaner.py)) + формулировка «исправляй искажения, не добавляй своё» сдерживает. |
| Очень длинный словарь | Обрезаем до N слов (как `_MAX_WORDS_PROMPT`). |

## Acceptance criteria

- Термины из словаря попадают в системный промпт всех LLM-бэкендов.
- На фразе-омофоне («запусти кампанию в рекламе») при наличии «кампания» в словаре LLM выдаёт верный вариант.
- Per-app промпт (концепт 07) и Command Mode продолжают работать — глоссарий добавляется, не затирает.
- При пустом словаре поведение очистки не меняется.
