# 37. Voice-first пакет: кодовые слова старт/стоп · «Причесать» · роли-адресаты

> Один цельный voice-first воркфлоу: **«талкер старт … [речь] … талкер стоп»** — диктовка хендс-фри; в начале можно назвать адресата («Глебу…», «начальнику Олегу…») → подбирается **роль** и её тон; кнопка **«Причесать»** на плашке результата переписывает текст лучше через ИИ под этот тон. Код — в `patch_2/` (standalone, основной код не тронут).

**Категория:** UX ввода + голосовое управление — высокая отдача.
**Эффект:** 🔥 заметно; риск низкий (всё аддитивно, LLM-агностично).
**Готовность концепта:** 🟢 High — логика в трёх протестированных модулях; нужен только вайринг (`patch_2/INTEGRATION.md`).
**Источник идеи:** запрос пользователя 2026-06-03. Ориентиры в коде: [wake_word.py](../wake_word.py) (21), [voice_commands.py](../voice_commands.py) (17), [modes.py](../modes.py) (07), [cleaner.py](../cleaner.py).

## A. Кодовые слова «талкер старт/стоп» — ✅ реализовано (`voice_gate.py`) 🟢

> Hands-free вернули (2026-06-04). Реализация — `voice_gate.py` (агент): гейт в `_on_continuous_segment`, тумблер `output.voice_gate`, переиспользует шим.

### Зачем
Полный hands-free: не тянуться к хоткею. Активация голосом уже есть (wake-word «эй талкер», концепт 21), но нет явной **остановки словом**. Кодовые слова дают пару старт/стоп прямо в речи.

### Подход
`patch_2/voice_triggers.py` — `VoiceTriggers.feed(chunk)` → `{started, stopped, active, text}`: детект фраз + вырезание их из текста; устойчиво к регистру/ё/пунктуации (свёртка длину сохраняет → индексы совпадают). Нужен источник continuous-текста: либо always-on STT, либо (рекомендую) поверх существующей wake-word-сессии.

### Архитектура
- `voice_triggers.py` (есть) → корень.
- `main.py` continuous-петля: на каждый чанк `feed()`; started→начать сегмент, text→вставлять пока active, stopped→финал (+звук stop из patch 1).
- `config.py` (опц.): `start_phrases/stop_phrases` в `OutputConfig`.

### Edge cases
| | |
|---|---|
| Кодовое слово в середине фразы | вырезается, остаток вставляется. |
| Ослышка («толкер») | в дефолтах есть варианты; список расширяем. |
| STT склеил «талкерстарт» | нужен разделитель; редко, можно добавить варианты. |
| Самозапуск от собственного TTS/звуков | кулдаун (как в wake_word). |

### Acceptance
«талкер старт … талкер стоп» открывает/закрывает диктовку; кодовые слова не попадают в текст.

## B. Роли-адресаты + выбор голосом 🟢

### Зачем
Тон зависит от того, КОМУ пишешь (друг / коллега / начальник / клиент). Это иная ось, чем per-app modes (те по приложению). Идея пользователя: **пресет тона создаётся ИИ** из короткого описания, а в начале диктовки называешь адресата.

### Подход
`patch_2/roles.py`:
- `Role(name, triggers, prompt, tone)` + `default_roles()`.
- `match_leading_recipient(text, roles)` — «Глебу, привет…» → (роль, «привет…»). Адресат в начале = селектор тона.
- `generate_role_prompt(desc, llm_call)` — «начальник Олег, формально, по делу» → системный тон (ИИ).
- роль дополняет per-app mode-промпт и задаёт тон для «Причесать».

### Архитектура
- `roles.py` → корень; `roles_settings_panel.py` (customtkinter) → корень, вставка в Settings одной строкой.
- `config.py`: `RoleConfig` + `roles: list[RoleConfig]` (мирроринг `ModeConfig`/`voice_commands`, save/load + дефолты из `default_roles`).
- `main.py._process`: `match_leading_recipient` перед cleanup → `system_prompt = mode_prompt + role.prompt`; роль запомнить для «Причесать».

### Edge cases
| | |
|---|---|
| «олег» ловит «олегу» | защищено `\b`-границей (тест есть). |
| Адресат = часть сообщения («Глебу, привет») | селектор вырезается (опц. оставлять как обращение). |
| Нет совпадения | роль None → обычный cleanup. |

### Acceptance
«Глебу, …» в начале → дружеский тон; роли создаются из описания через ИИ; список редактируется в Settings.

## C. Кнопка «Причесать» 🟢

### Зачем
Один клик — переписать надиктованное «набело» (чётче/грамотнее), сохранив смысл и факты. Движок готов (`cleaner.clean(system_prompt=...)` + gemma/OpenRouter).

### Подход / UI
`patch_2/polish.py` — `polish_text(text, llm_call, role_prompt)`. UI-эталон — `patch_2/polish_ui_preview.py`: на **плашке результата** рядом с «✎ Поправить» кнопка **«✨ Причесать»**; клик → «Причёсываю…» (кнопка занята) → текст заменяется → на ~6 c «↶ Вернуть» (откат). Тон берётся из активной роли (B).

### Архитектура
- `polish.py` → корень.
- `ui.py.PasteFallbackBubble`: кнопка + состояния + откат; polish в потоке.
- Результат прогонять через **faithguard** (анти-выдумки), как обычный cleanup.

### Edge cases
| | |
|---|---|
| LLM недоступен/таймаут | вернуть исходник (в `polish_text` уже try/except). |
| Модель «улучшила» = выдумала | faithguard откат; «↶ Вернуть». |
| Уже вставлено в приложение | заменять backspace×N (как streaming-ветка) или только в плашке + «вставить причёсанное». |

### Acceptance
«Причесать» переписывает текст, даёт откат, факты сохранены.

## Сложность
- A кодовые слова: ~3–5 ч (логика готова; работа — источник continuous-STT + петля).
- B роли: ~4–6 ч (config-мирроринг + матч в `_process` + панель готова).
- C причесать: ~2–3 ч (кнопка + состояния; движок и логика готовы).

## Решения (зафиксированы 2026-06-04)
1. **Hands-free — ВКЛючён** (пользователь сам пользуется). ✅ Реализован агентом в `voice_gate.py` (буфер между «талкер старт/стоп»), гейт в continuous-режиме, тумблер `output.voice_gate` в Settings (по умолчанию off). Переиспользует шим: роль/причеши/ввод применяются как в обычной диктовке.
2. Адресат — **вырезается** (`strip=True`), только при разделителе (`require_separator=True`).
3. «Причесать» — «причеши» в конце, ДО вставки, через настроенный движок очистки (твой OpenRouter API/локалка), вторым проходом.
4. «Ввод» — Enter после вставки (флаг `submit`).

Флоу: continuous (vad_segments) + `voice_gate=on` → «эй талкер»/хоткей → «талкер старт» → «Глебу, … причеши» → «талкер стоп, ввод» → вставка причёсанного + Enter. Либо обычный хоткей без гейта (роли/причеши/ввод работают и так).
