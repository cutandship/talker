# 06. Auto-learning Dictionary

> Когда юзер **исправляет** только что вставленный текст руками, фикс автоматически попадает в словарь. Через несколько недель Talker узнаёт твои имена/термины без явной настройки.

**Категория:** Tier 2 — zero-effort персонализация. Wispr Flow это рекламирует как killer-фичу.
**Готовность концепта:** 🟡 Medium — сама идея понятна, но требует осторожной эвристики, чтобы не научиться плохому.

---

## Зачем

Концепт **05 custom_vocabulary** требует, чтобы юзер сам додумался и добавил слова. На практике мало кто это делает.

Auto-learning замечает: «вот я вставил "клод", а юзер тут же стёр и набрал "Claude"» — и сам добавляет "Claude" в словарь.

Через 2 недели использования словарь содержит реальные термины юзера без единой настройки.

---

## Технический подход

### Сигнал «юзер исправил вставку»

Самый чистый сигнал — **сравнить вставленный текст с тем, что в итоге оказалось в поле через N секунд**. Но мы не имеем доступа к содержимому чужого поля.

Альтернативные сигналы:

**(a) Clipboard monitoring** — если юзер вскоре после вставки **сам копирует** что-то из той же области, и это «похоже» на наш текст, но изменено — это правка. Слабый сигнал.

**(b) UIA TextPattern read-back** — после вставки через UIA (концепт 04) можем периодически проверять содержимое focused element. Если оно изменилось — diff с нашим текстом. Дорого и ненадёжно.

**(c) Подтверждение через bubble** — bubble показывает 8 секунд. Добавляем кнопку «Не так?». Юзер кликает → открывается окно с raw текстом и cleaned, можно править. Изменения попадают в словарь. **Самый простой и контролируемый путь.**

**(d) Heuristic post-fact** — следующая диктовка, если содержит похожее по фонетике слово, **спрашиваем** "имели в виду X?". Тоже отдалённо.

**Выбор:** **(c) bubble feedback** как первая итерация. Простой, явный, юзер контролирует.

### UI bubble с правкой

В bubble (концепт реализован в `ui.py:PasteFallbackBubble`):

```
┌─ Готово • клик для копирования ───── ✕ ─┐
│                                          │
│ Привет, Клод, как дела сегодня?          │
│                                          │
│  ⎘  Копировать      ✎  Поправить ▾       │
└──────────────────────────────────────────┘
```

При клике «Поправить» — bubble расширяется в editable textarea + кнопка «Сохранить». Diff с оригиналом → выделяем изменения → отправляем в auto-learning.

### Diff → словарь

```python
def extract_corrections(original: str, corrected: str) -> list[tuple[str, str]]:
    """Returns [(wrong, right), ...]"""
    # difflib.SequenceMatcher на уровне слов
    sm = SequenceMatcher(a=original.split(), b=corrected.split())
    pairs = []
    for tag, i1, i2, j1, j2 in sm.get_opcodes():
        if tag in ("replace", "delete", "insert"):
            wrong = " ".join(original.split()[i1:i2])
            right = " ".join(corrected.split()[j1:j2])
            if wrong and right and _looks_like_name(right):
                pairs.append((wrong, right))
    return pairs
```

`_looks_like_name(s)` — эвристика:
- Начинается с заглавной (англ).
- Не stop-word.
- Длиннее 2 символов.
- Содержит латиницу.

### Хранение

`vocabulary.json` (отдельно от `config.toml`, потому что меняется автоматически и часто):

```json
{
  "words": {
    "Claude": {"added": "2026-05-28", "source": "auto", "uses": 7},
    "Anthropic": {"added": "2026-05-28", "source": "auto", "uses": 3},
    "Talker": {"added": "2026-05-20", "source": "manual", "uses": 12}
  }
}
```

Параметр `source` различает manual (через UI) vs auto. UI может показывать «✨ выучено автоматически» рядом с такими словами.

### Composing initial_prompt

В концепте 05 был наивный prompt. Здесь учитываем `uses` — часто используемые слова **впереди** в prompt (Whisper больше внимания на конец prompt'а).

## Edge cases

| Сценарий | Поведение |
|---|---|
| Юзер удалил весь текст | Не learning ничего. |
| Юзер дописал длинный кусок (целый абзац) | Не learning — это не правка, это продолжение. Только короткие правки слов считаем. |
| Юзер заменил "клод" на "клон" (тоже неверно) | К сожалению, learning сохранит "клон". Лечение: показывать learned word в UI с возможностью удалить. |
| Юзер удалил выученное слово в UI | Помечаем как `blacklisted`, больше не учим этому. |
| Auto-learning засрал prompt мусором | Каждые N=30 дней — sanity prune: слова с `uses=0` за период удаляются. |
| Конфликт manual vs auto на одно слово | Manual всегда сильнее. |

## Acceptance criteria

- Bubble в expand-режиме показывает редактор.
- Сохранение правки добавляет слова в vocabulary.
- Через 5 правок одного и того же слова Whisper стабильно его узнаёт.
- В Настройках видно, какие слова выучены автоматически.
- Можно удалить плохое выученное слово.

---

## Сложность

- ~5–7 часов, ~300 LOC.
- Большая часть — UI bubble в expand-режиме + diff-логика.
- Зависит от концепта 05 (custom vocabulary как база).

## Открытые вопросы

- Изучать ли правки в bubble автоматически, или **спрашивать**: «Запомнить "Claude" вместо "клод"?». Дефолт: спрашивать первые 5 раз, потом молча учить.
- Использовать ли фонетическое сравнение, чтобы понимать, что "клод" и "Claude" — это пара (а не "клод" и "stuff")? Soundex / Metaphone. Bonus в v2.
