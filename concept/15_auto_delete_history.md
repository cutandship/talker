# 15. Auto-delete history / privacy housekeeping

> Опция автоматически удалять записи истории через N дней или при достижении лимита N записей.

**Категория:** Tier 3 — privacy + housekeeping.
**Готовность концепта:** 🟢 High.

---

## Зачем

Сейчас `MAX_ENTRIES = 1000` в `history_mgr.py` — единственный механизм. История накапливается бессрочно, в ней может оказаться чувствительная информация (пароли, личное).

Aiko показал: «auto-delete after N days» — востребованная privacy-фича.

---

## Технический подход

### Опции

```toml
[history]
max_entries = 1000           # уже есть; держим
retention_days = 0           # 0 = бессрочно, иначе удалять старше N дней
on_quit_clear = false        # при выходе чистить всё
```

### Algorithm

При каждом `append()`:
1. Если `retention_days > 0`: удалить записи с `timestamp < now - retention_days`.
2. Применить `max_entries` (как сейчас).
3. Сохранить.

Дополнительно: при старте Talker — однократная очистка просроченных.

### UI

В Settings → блок «История»:

```
┌─ История ──────────────────────────────────────────────┐
│ Максимум записей:   [1000]                              │
│ Хранить:            ⦿ Бессрочно ⦾ ___ дней              │
│ [_] При выходе очищать историю                          │
│ [_] Хранить аудио вместе с текстом (опасно: место)     │
│                                                         │
│ Сейчас в истории: 247 записей, занято 1.2 МБ           │
│ [Очистить всю историю]                                  │
└────────────────────────────────────────────────────────┘
```

### Чувствительные слова / редакция

Опционально: чекбокс «Не сохранять записи, содержащие пароли / номера карт» — простой regex-фильтр. Если матч — record skipped (только в history, в clipboard вставляется).

Список паттернов (defaults):
- Card number: `\b\d{4}[\s-]?\d{4}[\s-]?\d{4}[\s-]?\d{4}\b`
- Password-like: words рядом с "пароль", "password" — fuzzy.

### Хранение аудио (опция)

Сейчас audio не сохраняется. Если юзер опт-инит:
- В `~/.talker/recordings/{timestamp}.wav` (или .opus для размера).
- Auto-delete по тем же правилам, что и history.

## Архитектура

**Изменения:**
- `history_mgr.py`:
  - `_prune_old()` метод, вызывается перед каждым `_save()`.
  - Поддержка `retention_days`, `on_quit_clear`.
- `config.py`:
  - `HistoryConfig` dataclass.
- `ui.py:SettingsWindow` — секция «История».
- `main.py:_quit` — если `on_quit_clear=True` → `history.clear()`.

## Edge cases

| Сценарий | Поведение |
|---|---|
| `retention_days = 0` и пользователь имел старые записи | Не трогаем. 0 = бессрочно. |
| Юзер вручную поменял timestamp в JSON | Удаляем на основании указанной даты. |
| Размер истории > 10 MB | Не страшно, JSON парсится быстро. |
| Аудио файлы накопились на 5 GB | Чистка по retention_days убирает. |
| Внешний backup (cloud sync) забрал history.json до удаления | Не наша забота. |

---

## Acceptance criteria

- При `retention_days = 7` записи старше недели автоматически исчезают.
- Лимит `max_entries` сохраняется как раньше.
- Чекбокс «При выходе очищать» работает (полная очистка на quit).
- UI показывает счётчик записей и занятое место.
- Можно вручную очистить всё одной кнопкой.
