# Как дообучить wake-модели до рабочего состояния

> Пошаговая инструкция. Написана так, будто ты никогда этого не делал.
> Цель: чтобы «Эй, Талкер» срабатывал на ТВОЙ голос и НЕ срабатывал на шум.
>
> **Суть в одном предложении:** записать свой реальный голос и реальный
> фоновый шум, подмешать к синтетическим данным, переобучить. Всё.

---

## Почему это нужно (на пальцах)

Сейчас модель училась только на «компьютерных голосах» (синтез). Реальный
твой голос через твой микрофон она не слышала — и путается, срабатывает на
шум. Мы дадим ей послушать настоящее: твой голос (примеры «да, это фраза») и
реальный фон твоей комнаты (примеры «нет, это не фраза»). После этого она
научится различать именно в твоих условиях.

Это называется **domain adaptation** — «подгонка под реальные условия».

---

## Что понадобится

- Этот компьютер (где уже всё установлено: папки `venv/`, `data/`, `out/`).
- Микрофон (тот же, которым пользуешься в Talker).
- ~1 час времени: 15 минут записывать, остальное — компьютер обучает сам.

> **Если ты на ДРУГОМ компьютере** (всё с нуля) — сначала раздел
> «Установка с нуля» в самом низу. Это долго (качать гигабайты). На этом
> компьютере всё уже есть — пропусти.

---

## Шаг 1. Записать свой голос (15 минут)

Открой PowerShell в папке `wake_training` и для каждой фразы запусти запись.
Начни с главной — «Эй, Талкер»:

```powershell
.\venv\Scripts\python.exe record_voice.py ey_talker
```

Скрипт скажет «Скажи: Эй, Талкер», отсчитает 3-2-1 и запишет. Повтори
**30 раз** (скрипт сам считает). Советы по записи он покажет, главное:
- говори **естественно**, как реально будешь пользоваться;
- **меняй** каждый раз чуть-чуть: громче/тише, быстрее/медленнее, повернись,
  отойди/подойди к микрофону;
- можно с лёгким фоном (телевизор, улица) — так даже лучше.

Потом то же для остальных трёх (если нужны все):
```powershell
.\venv\Scripts\python.exe record_voice.py talker_stop
.\venv\Scripts\python.exe record_voice.py stop_stop
.\venv\Scripts\python.exe record_voice.py stop_da
```

> Записи лягут в `out/<модель>/positive_train/` с именами `real_00000.wav`
> рядом с синтетическими. Старые НЕ затираются.

---

## Шаг 2. Записать реальный фон (10 минут) — ВАЖНО

Это половина успеха. Модель должна услышать, как звучит твоя комната, КОГДА
ты НЕ говоришь команду — чтобы не срабатывать на это.

Запиши 5-10 минут реального фона в файл. Самый простой способ — этой командой
(она пишет 10 минут с микрофона в один файл):

```powershell
.\venv\Scripts\python.exe -c "import sounddevice as sd, scipy.io.wavfile as w, numpy as np; print('Пишу 10 минут фона — занимайся обычными делами, говори, включи ТВ, НО не говори команды Talker...'); a=sd.rec(int(600*16000),samplerate=16000,channels=1,dtype='float32'); sd.wait(); w.write('data/background_16k/real_room.wav',16000,(np.clip(a.reshape(-1),-1,1)*32767).astype(np.int16)); print('Готово: data/background_16k/real_room.wav')"
```

Во время записи: **живи обычной жизнью** — поговори с кем-то, включи телевизор,
музыку, пусть шумит улица/кулер. **Только НЕ произноси сами команды** («эй
талкер» и т.п.) — иначе модель запутается.

> Файл ляжет в `data/background_16k/` — туда, откуда обучение берёт фон для
> подмешивания. Он автоматически попадёт в негативы.

---

## Шаг 3. Усилить подмешивание реального фона

Открой `configs/ey_talker.yml` (и остальные `configs/*.yml`) в Блокноте.
Найди строки:

```yaml
augmentation_rounds: 2
```

Поменяй на `4` — модель будет чаще «пачкать» чистые примеры реальным фоном,
лучше адаптируется. Сохрани.

---

## Шаг 4. Переобучить

Удали старые модели и сгенерированные признаки (примеры голоса НЕ трогаем!),
чтобы обучение пересчитало их с учётом новых записей:

```powershell
# из папки wake_training:
Remove-Item out\final\*.onnx -Force
Remove-Item out\*\*features*.npy -Force
Remove-Item out\*.onnx -Force
```

Запусти обучение (одна команда, делает всё, следит сама):

```powershell
powershell -ExecutionPolicy Bypass -File guard.ps1
```

Жди ~3-4 часа (4 модели по ~40 минут). Можно закрыть PowerShell — обучение
идёт в фоне, сторож следит. Проверить прогресс:
```powershell
Get-ChildItem out\final\*.onnx     # сколько моделей готово
```

---

## Шаг 5. Проверить НА ЖИВОМ микрофоне (не на синтезе!)

Когда в `out/final/` появятся 4 файла `.onnx`, проверь **вживую** —
это критично, синтетические тесты обманчивы:

```powershell
.\venv\Scripts\python.exe -c "import numpy as np, sounddevice as sd; from openwakeword.model import Model; m=Model(wakeword_models=['out/final/ey_talker.onnx'],inference_framework='onnx'); print('Слушаю 30 сек. Скажи Эй Талкер пару раз, потом помолчи/пошуми...'); import time; t=time.time(); peak=0; 
while time.time()-t<30:
 a=sd.rec(1280,samplerate=16000,channels=1,dtype='float32'); sd.wait(); s=float(list(m.predict((np.clip(a.reshape(-1),-1,1)*32767).astype('int16')).values())[0]);
 if s>0.5: print(f'  СРАБОТАЛО score={s:.2f}')
"
```

Смотри: когда говоришь «Эй, Талкер» — должно печатать «СРАБОТАЛО». Когда
молчишь/шумишь — НЕ должно. Если ложит на шум — добавь ещё фона (шаг 2) и
повтори обучение. Если не узнаёт тебя — добавь ещё своих записей (шаг 1).

---

## Шаг 6. Подключить в Talker

Когда модель проходит живой тест — пропиши её в `config.toml` Talker
(в корне проекта), секция `[wake]`:

```toml
model_path  = "C:\\Users\\user\\Desktop\\AI R&D\\Talker\\wake_training\\out\\final\\ey_talker.onnx"
threshold   = 0.7
stop_models = ["...\\out\\final\\talker_stop.onnx"]
```

Подбери `threshold` (порог) по живому тесту: ниже — срабатывает легче (но
больше ложных), выше — строже. Перезапусти Talker (`pythonw main.py`).

---

## Если совсем не выходит

Короткие фразы («стоп-стоп», «стоп-да») по своей природе ложат на похожие
звуки — это нормально, их лучше оставить ТЕКСТОВЫМИ (они и так работают в
Talker через распознавание речи). Сосредоточься на длинных: «Эй, Талкер» и
«Талкер, стоп» — они обучаются надёжно.

Альтернатива обучению: можно вообще не дообучать, а пользоваться стандартным
`hey_jarvis` (так сейчас и настроено) — он надёжный, просто не русский.

## Установка с нуля (если на новом компьютере)

Долго (качать ~15 ГБ). По порядку:

```powershell
# 1. Python 3.11, затем в папке wake_training:
python -m venv venv

# 2. библиотеки (torch под свою видеокарту — см. pytorch.org)
.\venv\Scripts\python.exe -m pip install torch --index-url https://download.pytorch.org/whl/cu124
.\venv\Scripts\python.exe -m pip install -e ./openWakeWord
.\venv\Scripts\python.exe -m pip install audiomentations torch-audiomentations speechbrain==0.5.14 mutagen torchinfo torchmetrics acoustics datasets==2.14.6 edge-tts scipy tqdm pyyaml soundfile av omegaconf pyroomacoustics onnxruntime onnx

# 3. шим вместо тяжёлого пакета acoustics (см. РАЗБОР_ПОЛЁТОВ.md)
.\venv\Scripts\python.exe make_acoustics_shim.py

# 4. данные (негативы + модели препроцессора):
.\venv\Scripts\python.exe download_features.py
.\venv\Scripts\python.exe make_partial_acav.py    # если ACAV не докачался целиком
# конвертация негативов в float32:
.\venv\Scripts\python.exe -c "import numpy as np; from numpy.lib.format import open_memmap; s=np.load('data/acav_partial.npy',mmap_mode='r'); d=open_memmap('data/acav_partial_f32.npy',mode='w+',dtype=np.float32,shape=s.shape); [d.__setitem__(slice(i,i+50000), s[i:i+50000].astype(np.float32)) for i in range(0,len(s),50000)]; d.flush()"

# 5. проверка синтеза и запуск:
.\venv\Scripts\python.exe smoke_tts.py
powershell -ExecutionPolicy Bypass -File guard.ps1
```

> ⚠️ Все патчи под Windows уже внесены в наш клон `openWakeWord/`. Если ставить
> openWakeWord заново из интернета — патчи (#3-#6 из РАЗБОР_ПОЛЁТОВ.md) слетят,
> придётся вносить заново. Лучше скопировать наш пропатченный `openWakeWord/`.
