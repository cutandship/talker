# Обучение голосовых команд Talker (wake-слова)

Эта папка — **полностью рабочая лаборатория** для обучения голосовых
команд-активаторов («Эй, Талкер», «Талкер, стоп» и т.п.) для Talker.
Здесь скрипты, конфиги, готовые модели и подробная документация — чтобы можно
было вернуться через полгода и всё повторить с нуля.

## Что это вообще такое

«Wake-слово» (wake word) — это короткая фраза, которую программа постоянно
слушает в фоне и, услышав, что-то делает. Как «Привет, Сири» или «Окей, Гугл».

В Talker таких фраз четыре:

| Фраза | Что делает |
|---|---|
| **«Эй, Талкер»** | начать диктовку без рук (wake) |
| **«Талкер, стоп»** | закончить диктовку |
| **«стоп-стоп»** | закончить диктовку (короткий вариант) |
| **«стоп-да»** | закончить **и** нажать Enter (отправить) |

Каждая фраза — это отдельная маленькая нейросеть (файл `.onnx`, ~840 КБ).
Технология — [openWakeWord](https://github.com/dscripka/openWakeWord)
(открытая, бесплатная).

## ⚠️ ТЕКУЩИЙ СТАТУС (читать обязательно)

**Модели обучены, лежат в `out/final/`, но В ПРОДАКШЕНЕ НЕ ИСПОЛЬЗУЮТСЯ.**

Почему: они обучены на **синтетической речи** (компьютерные голоса, TTS).
На таких же синтетических тестах они работают идеально (срабатывают на свою
фразу с уверенностью 100%, на чужие — 0%). **Но на реальном микрофоне в
реальной комнате они ложно срабатывают на фоновый шум.**

Это называется **domain gap** («разрыв доменов») — модель учили на одном типе
звука (чистый синтез), а используют на другом (живой микрофон + шум комнаты).
Классическая проблема, не баг в коде.

**Сейчас в Talker работает:**
- wake — стандартная модель `hey_jarvis` (обучена профессионалами на реальных
  данных, не ложит);
- стопы — текстовые («стоп-стоп», «стоп-да» распознаются обычным
  распознаванием речи, не нейросетью-активатором).

**Чтобы наши модели заработали** — нужно дообучить их на ЗАПИСЯХ ТВОЕГО голоса
с ТВОЕГО микрофона. Как — в [`docs/КАК_ДООБУЧИТЬ.md`](docs/КАК_ДООБУЧИТЬ.md).

## Структура папки

### Скрипты (это в git, это и есть «исходники»)

| Файл | Что делает |
|---|---|
| `gen_samples.py` | генерирует синтетические примеры фраз (TTS-голоса Silero + edge-tts) |
| `prepare_data.py` | готовит данные аугментации (эхо комнат + фоновый шум) |
| `run_night.py` | **главный оркестратор**: запускает всю цепочку обучения 4 моделей |
| `validate_e2e.py` | честная проверка: синтезирует фразу → прогоняет через модель → меряет, срабатывает ли |
| `guard.ps1` | сторож: следит, что обучение не упало, перезапускает |
| `configs/*.yml` | настройки обучения каждой модели (какая фраза, сколько шагов) |
| `download_features.py` | качает обучающие данные (негативы ACAV, валидация) |
| `make_partial_acav.py` | спасает недокачанный большой файл негативов |
| `smoke_tts.py` | быстрая проверка, что синтез голоса работает |
| `piper_stub/` | заглушка (официальный генератор работает только на Linux) |

### Что НЕ в git (большое, качается/генерируется заново)

| Папка | Размер | Что это |
|---|---|---|
| `venv/` | ~6 ГБ | окружение Python (torch, openwakeword и т.д.) |
| `data/` | ~10 ГБ | обучающие данные (негативы, шумы, эхо) |
| `out/` | ~2 ГБ | сгенерированные примеры фраз + готовые модели |
| `tts_cache/` | ~0.3 ГБ | кэш синтезированных голосов |
| `openWakeWord/` | — | клон библиотеки openWakeWord (с нашими патчами!) |

> **Важно:** в `openWakeWord/` лежат НАШИ ПАТЧИ (6 правок под Windows). Если
> переустанавливать — патчи слетят. Они описаны в
> [`docs/РАЗБОР_ПОЛЁТОВ.md`](docs/РАЗБОР_ПОЛЁТОВ.md), раздел «Патчи».

### Готовые модели

`out/final/` — четыре `.onnx` файла. Рабочие (на синтезе), но с domain gap.
Подключаются в Talker одной строкой в `config.toml`, секция `[wake]`:
```toml
model_path  = "...\\wake_training\\out\\final\\ey_talker.onnx"
stop_models = ["...\\wake_training\\out\\final\\talker_stop.onnx"]
```

## Как запустить обучение заново (если данные и venv на месте)

```powershell
# из папки wake_training, ОДНА команда — она сделает всё:
powershell -ExecutionPolicy Bypass -File guard.ps1
```
`guard.ps1` запустит `run_night.py`, который: подготовит данные → сгенерирует
примеры → обучит 4 модели → положит в `out/final/`. Следит сам, перезапускает
при сбоях. Занимает ~3-5 часов на RTX 4090.

Потом проверка:
```powershell
.\venv\Scripts\python.exe validate_e2e.py
```

**Если venv/данных нет** (свежая машина) — см.
[`docs/КАК_ДООБУЧИТЬ.md`](docs/КАК_ДООБУЧИТЬ.md), раздел «Установка с нуля».
